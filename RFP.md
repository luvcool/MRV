# HKMC 2D Barcode Web Scanner

## Request for Proposal (RFP)

## 1. 프로젝트 개요

본 프로젝트의 목적은 HKMC(Hyundai / Kia Motor Company) 부품용 Data
Matrix 2D 바코드를 스캔하고 해석하는 모바일 웹 애플리케이션을 개발하는
것이다.

환경: - iPhone Safari - Android Chrome - 모바일 웹(PWA) - Cloudflare
Pages 배포

모든 바코드 분석은 브라우저 내부에서 수행한다. 이미지 데이터는 서버로
전송하지 않는다.

------------------------------------------------------------------------

# 2. 시스템 목표

시스템은 다음 기능을 수행해야 한다.

1.  모바일 카메라로 Data Matrix 바코드를 스캔
2.  QR 코드 보조 지원
3.  HKMC DI(Data Identifier) 규격 파싱
4.  파싱 데이터를 표 형태로 표시
5.  원문 바코드 문자열 표시
6.  최근 스캔 이력 저장

------------------------------------------------------------------------

# 4. HKMC 2D Barcode 구조

데이터는 DI(Data Identifier) 기반이다.

각 필드는 다음 형식을 가진다.

DI + Data

데이터는 GS(Group Separator)로 구분된다.

예시:

\]\>RS06 GS Vxxxx GS Pxxxxxxxxxx GS Sxxxx GS Exxxxxxxx GS Txxxxxxxxxxxx
GS 1xxxx GS Mx GS Cxxxxx RS EOT

------------------------------------------------------------------------

# 5. Data Identifier 규칙

  DI   의미          길이
  ---- ------------- -------------
  V    업체 코드     4 byte
  P    부품 번호     10\~15 byte
  S    서열 코드     1\~8 byte
  E    EO 번호       8\~9 byte
  T    추적 코드     14\~36 byte
  1    특이 정보     1\~40 byte
  M    초도품 구분   1 byte
  C    업체 영역     1\~50 byte

------------------------------------------------------------------------

# 6. Trace Code (T) 구조

예:

T150520S1B2A0476217

구성

  위치    의미
  ------- -----------------
  1\~6    생산일자 YYMMDD
  7\~10   4M 정보
  11      Lot 관리 방식
  12\~N   제조 시리얼

Lot 방식

A = 개별 Serial 관리\
@ = 다수 Lot 관리

------------------------------------------------------------------------

# 7. Multi-Part 규칙

바코드 문자열 안에 "\#" 문자가 포함된 경우 이는 여러 부품이 하나의
바코드에 포함된 경우이다.

예

P25304DN000#P25305DN000

동작

1.  # 기준으로 분리

2.  각각 개별 Part Number로 인식

3.  UI에서 리스트로 표시

------------------------------------------------------------------------

# 8. 바코드 파싱 규칙

파싱은 반드시 DI 기반으로 수행한다.

필드 순서를 가정하면 안 된다.

알고리즘

1.  RS 제거
2.  GS 기준 split
3.  DI 식별
4.  데이터 저장
